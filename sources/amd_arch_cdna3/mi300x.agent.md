# MI300X Agent Notes

This file is a MI300X-only knowledge note for agents.
It intentionally keeps scope narrow and avoids cross-SKU comparisons.

## Scope

- Product focus: AMD Instinct MI300X.
- Architecture context: AMD CDNA 3 concepts that directly describe MI300X.
- Excludes: references to other product variants and cross-product benchmark claims.

## MI300X Hardware Highlights

- Chiplet package organized around accelerator compute dies and I/O infrastructure.
- Up to 8 accelerator complex dies (XCDs) in the package design context.
- High-bandwidth memory integration with large on-package bandwidth.
- Coherent high-speed fabric links for scale-up multi-accelerator systems.

## Compute Model (MI300X-Relevant)

- Compute Units (CUs) are the core execution blocks for scalar/vector/matrix work.
- Matrix and vector pipelines are optimized for HPC and AI throughput.
- Sparse-data acceleration support is part of the CDNA 3 generation features.
- Cache/memory hierarchy is redesigned to scale with higher compute density.

## Memory and Data Movement

- MI300X uses on-package HBM memory with high aggregate throughput.
- Data movement efficiency depends on locality, cache usage, and coalesced accesses.
- Work partitioning should minimize cross-partition traffic for better scaling.

## Software and ISA Pointers

- Low-level instruction definitions are in `../amdgpu_isa_cdna3.xml`.
- Use ROCm stack components for compiler/runtime/library support on MI300X.
- For kernel tuning, prefer occupancy, memory coalescing, and wave-level efficiency checks.

- Treat this file as the primary text context for MI300X product notes.
- If additional sources are added later, keep them MI300X-only.
